Core

Zero-Boilerplate Schemas

The @Schema macro generates conformances, initializers, and row mapping at compile time. Write a struct. Get a model.

@Schema("posts")
struct Post {
    @ID var id: UUID
    @Column var title: String
    @ForeignKey var authorId: UUID
    @Timestamp var createdAt: Date
    @BelongsTo var author: User?
}
Queries

Composable Query Builder

Query<T> is a value type. Every method returns a new query. Branch it, reuse it, compose freely.

Safety

No Silent N+1

Unpreloaded relationships assert in debug builds. You'll know before production does.

Flexibility

Generic Primary Keys

@ID<UUID>, @ID<Int>, or @ID<String>. Foreign keys match the referenced type automatically.

Schema

Real SQL Migrations

Plain .sql files with -- migrate:up / -- migrate:down. No ORM DDL to learn. Your whole team can review them.

-- migrate:up
CREATE TABLE "users" (
    "id"    UUID PRIMARY KEY
            DEFAULT gen_random_uuid(),
    "name"  TEXT NOT NULL,
    "email" TEXT NOT NULL UNIQUE
);

-- migrate:down
DROP TABLE "users";
Encoding

Encodable Schemas

The @Schema macro auto-generates Encodable conformance. Pass models straight to JSONEncoder — no manual dictionary mapping. Snake_case keys by default, @Column("custom") overrides them, and unloaded relationships are omitted automatically.

// Before — manual dictionary mapping
donuts.map { d in
    ["id": "\(d.id)",
     "name": d.name,
     "price": "\(d.price)"]
}

// After — direct encoding
try JSONEncoder().encode(donuts)
Code

See it in action.

Composable Queries

Query<T> is a value type. Build a base, branch into different results. The original is never mutated.

Queries.swift
let base = repo.query(User.self)
    .where { $0.isActive == true }

// Branch 1: recent active users
let recent = try await base
    .orderBy({ $0.createdAt }, .desc)
    .limit(10)
    .all()

// Branch 2: count of same base query
let total = try await base.count()

// Expressive operators
let results = try await repo.query(User.self)
    .where { $0.email.iContains("@gmail.com") }
    .where { $0.age.between(18, and: 65) }
    .where { $0.role.in(["admin", "moderator"]) }
    .orderBy({ $0.name }, .asc)
    .all()

Transactions

Full CRUD inside transactions. Automatic rollback on error. The repo conforms to the same Repo protocol — works identically inside and outside.

Transactions.swift
let (user, profile) = try await spectro.transaction { repo in
    let user = try await repo.insert(
        User(name: "Alice", email: "alice@example.com")
    )

    let profile = try await repo.insert(
        Profile(bio: "Hello!", userId: user.id)
    )

    // Full query builder inside transactions
    let count = try await repo.query(User.self)
        .where { $0.isActive == true }
        .count()

    return (user, profile)
}
// Automatic ROLLBACK on any thrown error

Relationship Preloading

Batch-load relationships in separate queries. Two queries total, not N+1. Composes with every other query method.

Preloading.swift
let users = try await repo.query(User.self)
    .where { $0.isActive == true }
    .preload(\.$posts)
    .preload(\.$profile)
    .preload(\.$tags)       // ManyToMany
    .orderBy({ $0.name }, .asc)
    .limit(50)
    .all()

// Access with confidence
for user in users {
    print(user.$posts.value)    // [Post] — loaded
    print(user.$profile.value)  // Profile? — loaded
}

// Without preload? Debug assertion:
// "HasMany<Post> accessed before loading.
//  Use .preload(\.$posts) in your query."

Aggregates & GROUP BY

SUM, AVG, MIN, MAX, COUNT. Grouped aggregates with HAVING. Type-safe results.

Aggregates.swift
let revenue = try await repo.query(Order.self)
    .where { $0.status == "completed" }
    .sum { $0.amount }

let avg = try await repo.query(Order.self).avg { $0.amount }

// Grouped with HAVING
let byRegion = try await repo.query(Order.self)
    .groupBy({ $0.status }, { $0.region })
    .having { $0.amount > 100 }
    .groupedSum { $0.amount }

// Field selection (no full model load)
let names: [String] = try await repo.query(User.self)
    .select { $0.name }
    .all()

Encodable Schemas

@Schema auto-generates Encodable conformance. JSON keys are snake_case by default. @Column("custom") overrides the key. Unloaded relationships are omitted from output. Opt out with @Schema("secrets", encodable: false).

Encoding.swift
@Schema("donuts")
struct Donut {
    @ID var id: UUID
    @Column var name: String
    @Column("display_price") var price: Decimal
    @Timestamp var createdAt: Date
    @BelongsTo var shop: Shop?
}

// Direct encoding — no manual mapping
let donuts = try await repo.query(Donut.self).all()
let json = try JSONEncoder().encode(donuts)
// → [{"id":"...","name":"Glazed","display_price":"3.50",
//     "created_at":"2026-03-28T..."}]
//   ↑ shop key omitted — not preloaded

// Preload to include relationships
let withShop = try await repo.query(Donut.self)
    .preload(\.$shop)
    .all()
let full = try JSONEncoder().encode(withShop)
// → [{"id":"...","name":"Glazed",...,
//     "shop":{"id":"...","name":"Sweet Spot"}}]

Get started

Three steps. That's it.

01

Install

Terminal
mint install Spectro-ORM/Spectro
02

Add dependency

Package.swift
dependencies: [
    .package(url: "https://github.com/Spectro-ORM/Spectro.git", from: "1.0.0")
]

// Add to your target:
.product(name: "SpectroKit", package: "Spectro")
03

Define & query

App.swift
import Spectro

@Schema("users")
struct User {
    @ID var id: UUID
    @Column var name: String
    @Column var email: String
}

let spectro = try Spectro.fromEnvironment()
let repo = spectro.repository()

let user = try await repo.insert(
    User(name: "Alice", email: "alice@example.com")
)

let users = try await repo.query(User.self)
    .where { $0.email.endsWith("@example.com") }
    .all()
